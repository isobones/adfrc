# Security Policy

## Supported Versions

Only the latest release of ADF: Re-Cut receives security updates. The current
release is available on the [Steam Workshop](https://steamcommunity.com/sharedfiles/filedetails/?id=2971219389).

## Reporting a Vulnerability

If you discover a security issue in ADF: Re-Cut, please report it privately
through one of the following channels:

- **Email:** adfrecut+security@gmail.com
- **GitHub Private Vulnerability Reporting:** use the "Report a vulnerability"
  button on the [Security tab](https://github.com/IsoBones/ADFRC/security)
  of this repository

Please do **not** report security issues via public GitHub issues, pull requests,
or the Discord server.

When reporting, include where possible:

- A description of the issue and its potential impact
- Steps to reproduce
- Affected files, configs, or scripts
- Mod version (Workshop or GitHub release tag)

You can expect an initial response within a reasonable timeframe. Once the issue
is confirmed and resolved, we will coordinate disclosure with you.

## Scope

In scope:

- Malicious or exploitable SQF scripts within the mod
- Config issues that could be abused on multiplayer servers
- Issues affecting client or server stability that have a security dimension

Out of scope:

- General bugs and balance issues — please use [GitHub Issues](https://github.com/IsoBones/ADFrecut/issues)
- Issues in dependencies (e.g. CBA_A3) — report to those projects directly
- Issues in Arma 3 itself — report to Bohemia Interactive
